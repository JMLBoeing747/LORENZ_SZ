Déchiffrer un message

Dans cette page d'aide, vous trouverez toutes les informations utiles à savoir pour déchiffrer un message avec LORENZ.

Étapes pour déchiffrer un message à partir du texte copié

  1. Copiez le message à déchiffrer. Assurez-vous qu'il s'agit bel et bien d'un chiffrement de LORENZ
  2. Dans le menu principal, choisir l'option 2 : Déchiffrement en appuyant sur la touche 2 du clavier ou du pavé numérique
  3. Lorsque l'instruction fera son apparition, collez le message à déchiffrer. Pour terminer, appuyez sur CTRL + D (^D).
  4. LORENZ déchiffre le message et vous l'affiche

Étapes pour déchiffrer un message à partir d'un fichier

  1. Assurez-vous que le fichier en question est placé dans votre répertoire des chiffrements, tel que vous l'avez spécifié en options. Si vous ne l'avez pas encore spécifié, consultez cette page.
  2. Dès les premières lignes de déchiffrement, tapez "FILE:" suivi du nom du fichier (sans l'extension)
  3. Appuyez sur CTRL + D pour terminer.
  4. LORENZ déchiffre le message provenant du fichier et vous l'affiche

Possibles messages d'erreur

LORENZ WARNING MSG: THIS MESSAGE IS PRIVATELY DESTINATED TO ANOTHER USER
-> Signifie que ce message est privé, donc qu'il ne vous est pas destiné et ne peut alors être affiché.
Pour remédier à cela, demandez à votre correspondant de vous inclure dans la communication privée, si cela est possible, bien entendu.
ATTENTION LE MESSAGE PEUT CONTENIR DES DÉFORMATIONS DUE À UNE MODIFICATION ACCIDENTELLE OU MALINTENTIONNÉE DE VALEURS DANS LE CHIFFREMENT
-> Signifie que ce message a été déformé (soit par une mauvaise transcription du chiffrement de la part de l'émetteur ou du récepteur, ou bien par une interception qui aurait modifié accidentellement ou intentionnellement le chiffrement) et peut donc contenir des erreurs de déchiffrement.
Pour remédier à cela, recopiez et recollez le message chiffré de la source et validez.
Si l'erreur persiste, demandez à votre correspondant de vous renvoyer un nouveau message chiffré avec le même contenu.
TYPING BREAK ERROR : SPACES DETECTED.
-> Signifie que ce message possède des espaces, et donc qu'il a été mal transcrit.
Pour remédier à cela, essayez de recopier et recoller le message chiffré en vous assurant qu'il n'y ait pas d'espace présents, surtout dans les bords de l'affichage console.
Si l'erreur persiste, demandez à votre correspondant de vous renvoyer un nouveau message chiffré avec le même contenu.
ERREUR FATALE CODE 55 : Le chiffrement n'est pas authentique.
-> Signifie que le déchiffrement n'a pu être réalisé car la clé de chiffrement s'est avérée invalide.
Pour remédier à cela, demandez à votre correspondant de vous renvoyer un nouveau message chiffré avec le même contenu.
Si l'erreur persiste, il s'agit probablement d'une erreur logique au sein de l'application. Consultez la page FAQ du Dépannage au sein de la rubrique concernée.
