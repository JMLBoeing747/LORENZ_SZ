Foire aux questions

Bienvenue à la foire aux questions. Cette page contient diverses questions qui pourraient vous concerner, accompagnées de leurs réponses et solutions envisageables. Notez bien qu’il s’agit d’une liste non exhaustive et qu’il peut y en avoir de nouvelles au fil de futures mises à jour. Ainsi, il est possible que votre question ne s’y retrouve pas, ou que la réponse recherchée ne vous satisfasse nullement. De cela, je vous recommande de consulter la page Soutien technique pour mieux vous aider. En agissant ainsi, vous contribuerez à enrichir cette FAQ, et ainsi potentiellement d'aider de futurs utilisateurs.

Pourquoi LORENZ me dit que je n'ai pas les droits d'accès à ce programme ?

R : Deux scénarios sont possibles :

  • Aucune clé de produit valide n'a été détectée.
    • Solution : Demandez à votre fournisseur LORENZ de vous générer une nouvelle clé de produit, puis insérez-le dans le répertoire principal de l'application.
  • Un des fichiers .LZI a été corrompu.
    • Solution : Voyez dans le dossier LZPARAMS, du répertoire principal du programme, si le fichier USERLOG.LZI s'y trouve. Si oui, supprimez-le définitivement. Ensuite, redemandez à votre fournisseur LORENZ de vous générer une nouvelle clé de produit et refaites le processus d'identification.

J'ai inséré la clé de produit, mais LORENZ ne le détecte pas. Pourquoi ?

R : Trois scénarios sont possibles :

  • La clé de produit n'est pas dans le bon dossier.
    • Solution : Vérifiez si la clé est insérée dans le dossier du répertoire principal du programme. Si la clé ne s'y trouve pas, veillez à l'insérer à cet endroit.
  • Le nom du fichier contenant la clé de produit n'est pas PRDCTKEY.LKI
    • Solution : Si tel est le cas, simplement renommer le fichier par PRDCTKEY.LKI. Assurez-vous que les extensions de fichiers soient affichées pour vérifier si l'extension est la bonne.
      Pour afficher les extensions de fichiers, cliquez ici pour voir une petite animation créée pour la circonstance.
  • Les données de la clé de produit sont corrompues.
    • Solution : Supprimez le fichier de la clé de produit (PRDCTKEY.LKI). Ensuite, redemandez à votre fournisseur LORENZ de vous générer une nouvelle clé de produit et refaites le processus d'identification.

Mon correspondant ne peut pas déchiffrer le message. Que s'est-il passé ?

R : Il est fort probable que le chiffrement est corrompu par une modification de certaines valeurs. En effet, la modification d'une seule lettre peut rendre le message illisible, voir même indéchiffrable. Une erreur ou un avertissement est toujours levé lorsque LORENZ a décelé un problème durant l'opération de déchiffrement.

Solutions :

  1. La plus rapide et la plus efficace consiste à renvoyer le message.
  2. Si cela ne règle pas le problème, veillez à vérifier si vous recopiez correctement le message chiffré et si votre correspondant fait de même. Pour cela, copiez le chiffrement, accédez à l'interface de déchiffrement et collez-le. Après validation, vérifiez si votre message est correctement déchiffré en sortie. Si cela ne change rien, demandez à votre correspondant de faire de même afin de vérifier si le problème n'est pas de son côté.
  3. Si le problème n'est toujours pas réglé, il se peut qu'il s'agisse d'un problème d'encodage dans l'environnement de transfert. Essayez de changer d'environnement (Messenger, epad.recit.org, etc.).
  4. Si le problème persiste, il pourrait s'agir d'une erreur informatique dans le programme. De cela, contactez votre fournisseur LORENZ afin qu'il puisse vous fournir un correctif dans les plus brefs délais.

Pourquoi mon solde de Coins accumulés ne s'affiche pas ou ne donne pas le bon montant ?

R : Il est fort probable que le fichier COINSREC.LZI a été corrompu. Si vous avez déjà gagné une partie, le montant gagné sera immédiatement affiché en bas du menu principal. Si, après une nouvelle connexion, le montant ne s'affiche plus, LORENZ a détecté une erreur dans le chiffrement de ce fichier et a donc réinitialisé le montant. Malheureusement, vous ne pouvez pas inscrire le montant gagné dans le chiffrement (sauf si vous connaissez l'algorithme évidemment), et n'avez donc le choix que de recommencer à zéro.

Pourquoi LORENZ n'affiche pas correctement mon pseudo que j'ai personnalisé ?

R : Il est fort probable que le fichier LORENZ.INI a été corrompu. La lecture du fichier a alors été mal réalisée.

Solutions :

  1. La plus rapide et la plus simple consiste à réaccéder au menu de modification du pseudo et à y réécrire le pseudo choisi. Fermez ensuite le programme et rouvrez-le.
  2. Si cela ne règle pas le problème, supprimez le fichier LORENZ.INI, situé dans le dossier LZPARAMS du répertoire principal de l'application. Refaites ensuite la procédure 1.
  3. Si cela ne règle pas le problème, supprimez le fichier PARAMS.INI, s'il est présent, situé dans le dossier LZPARAMS du répertoire principal de l'application. Refaites ensuite la procédure 1.
  4. Si le problème persiste, il pourrait s'agir d'une erreur informatique dans le programme. Contactez votre fournisseur LORENZ afin qu'il puisse vous fournir un correctif dans les plus brefs délais.

Comment accéder au répertoire principal de l'application ?

R : Accédez à la commande Exécuter de Windows, et ce en appuyant simultanément Win + R, et entrez %LORENZPATH% dans la zone prévue à cet effet (la casse n'importe pas). L'explorateur de fichiers ouvrira le dossier correspondant au répertoire principal de l'application, qui se nomme "LORENZSZ".

J'ai une erreur du type "Windows ne trouve pas '%LORENZPATH%'. Vérifiez que vous avez entré le nom correctement, puis réessayez." Qu'est-ce que cela signifie ?

R : Cela signifie que la variable %LORENZPATH% (la casse n'importe pas d'ailleurs) n'a pas été initialisée dans votre système, notamment après une première installation de l'application.

Solutions :

  1. Redémarrez l'ordinateur.
  2. Si le problème persiste, désinstallez LORENZ, puis réinstallez-le. Ceci permettra de refaire correctement le processus d'initialisation de la variable.

Pourquoi l'aide ne s'ouvre pas lorsque je désire y accéder ?

R : Deux scénarios sont possibles :

  • Le manuel d'aide LZHELP.CHM n'est pas présent dans le répertoire principal de l'application.
    • Solution : Demandez à votre fournisseur LORENZ de vous fournir un exemplaire du manuel d'aide LZHELP.CHM. Lorsque ceci sera fait, placez-le directement dans le répertoire principal de l'application LORENZSZ.
  • Le programme d'aide HH.EXE n'est pas présent sur votre système ou est malfonctionnel.
    • Ceci est un cas spécifique à Windows et vous devez vous référer au support technique de ce dernier afin qu'il puisse mieux vous orienter à ce sujet.
